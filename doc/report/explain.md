# WebUI 右侧参数面板详解

> 这份文档用最简单直白的话，解释 WebUI 右侧面板里每个指标、图表、参数都是什么意思 👇

---

## 1️⃣ 模型性能指标（最上面的 4 个卡片）

这是模型的"成绩单"，帮你判断模型好不好。

| 指标 | 什么意思？ | 怎么看好坏？ |
|------|-----------|-------------|
| **mAP@50** | IoU≥0.5 时的平均精度。可以理解为：在比较宽松的匹配标准下，模型有多少把握猜对？ | 越高越好，>80% 算不错 ✅ |
| **mAP@50-95** | IoU 从 0.5 到 0.95 的平均精度。综合了各种严格程度的匹配标准。 | 越高越好，这是更严格的标准 |
| **Precision（精确率）** | 模型说"这是石头"的时候，有多少次真的石头？ | 越高 = 越少误报（别把剪刀当石头） |
| **Recall（召回率）** | 所有石头里，模型找到了多少？ | 越高 = 越少漏检（别把石头漏掉了） |

> 💡 记忆小技巧：
> - Precision = "准不准"（宁缺毋滥）
> - Recall = "全不全"（宁可错杀不可漏过）

## 2️⃣ 训练结果图（results.png）

### 这是什么？
展示训练过程中各项**损失（Loss）**和**评估指标**的变化趋势。

### 坐标轴含义
| 轴 | 意思 |
|----|------|
| **X轴（Epoch）** | 训练轮数。第1轮、第2轮...第100轮 |
| **Y轴** | 损失值或指标值 |

### 曲线含义
| 曲线 | 名字 | 什么意思？ | 怎么看？ |
|------|------|------------|----------|
| 🔴 **train_loss** | 训练损失 | 模型在训练集上犯错的程度 | 应该一直下降 📉 |
| 🟡 **val_loss** | 验证损失 | 模型在验证集上犯错的程度 | 应该下降后平稳，如果上升说明过拟合了 ⚠️ |
| 🟢 **mAP50** | mAP@50 | 检测精度 | 应该一直上升 📈 |

### 怎么看模型有没有问题？
- ✅ 正常：train_loss 和 val_loss 都下降，最后平稳
- ⚠️ 过拟合：train_loss 下降，但 val_loss 上升 → 模型背答案了
- ⚠️ 欠拟合：两个损失都不下降 → 模型没学到东西

## 3️⃣ 混淆矩阵（confusion_matrix.png）

### 这是什么？
就是一个表格，告诉你模型对每个类别的预测结果。

### 坐标轴含义
| 轴 | 意思 |
|----|------|
| **X轴（Predicted）** | 模型预测的类别 |
| **Y轴（True）** | 实际的真实类别 |
| **颜色深浅** | 样本数量，越深越多 |

### 怎么看？
- 📍 **对角线**（左下到右上）：代表预测正确，颜色越深越好
- ❌ **非对角线**：代表预测错误，模型把这个类别误判成了另一个

### 例子
假设你是"石头-剪刀-布"三分类：
- 石头行：石头 95 个，布 3 个，剪刀 2 个 → 大部分石头被正确识别
- 剪刀行：石头 5 个，剪刀 90 个，布 5 个 → 剪刀和石头容易混淆

## 4️⃣ 归一化混淆矩阵（confusion_matrix_normalized.png）

### 这是什么？
和上面一样的混淆矩阵，但是把数字变成**比例**（0-1 之间），方便比较。

### 坐标轴含义
| 轴 | 意思 |
|----|------|
| **X轴（Predicted）** | 模型预测的类别 |
| **Y轴（True）** | 实际的真实类别 |
| **颜色深浅** | 比例值 0-1 |

### 怎么看？
- 对角线数值**接近 1** → 该类别预测准确
- 某一行某个非对角线位置数值高 → 这个真实类别经常被误判成对应类别

---

## 5️⃣ Precision-Recall 曲线（pr_curve.png）

### 这是什么？
展示在不同**置信度阈值**下，Precision 和 Recall 的权衡关系。

> 置信度阈值可以理解为：模型要多确定才说"这是XX"？

### 坐标轴含义
| 轴 | 意思 |
|----|------|
| **X轴（Recall）** | 召回率，找到了多少真实的目标 |
| **Y轴（Precision）** | 精确率，预测准不准 |

### 怎么看？
- 🎯 曲线越靠近**右上角**越好（既准确又全面）
- 不同颜色代表不同类别
- 理想情况：高 Precision 同时高 Recall

### PR 曲线 vs mAP
- PR 曲线下方的面积 = AP（Average Precision）
- mAP = 所有类别 AP 的平均值
- 所以 PR 曲线越饱满，mAP 越高

## 6️⃣ 训练曲线（Interactive Chart）

这是 WebUI 里可以交互的曲线图，用 Recharts 库画的。

### 坐标轴含义
| 轴 | 意思 |
|----|------|
| **X轴（Epoch）** | 训练轮数 |
| **Y轴** | 损失值/指标值 |

### 曲线含义
| 曲线颜色 | 意思 |
|----------|------|
| 🔴 红线 | train_loss（训练损失） |
| 🟡 黄线 | val_loss（验证损失） |
| 🟢 绿线 | mAP@50（检测精度） |

### 交互功能
- 🖱️ 鼠标悬停可以看到每个点的具体数值
- 📊 点击图例可以显示/隐藏某条曲线

## 7️⃣ 训练批次示例（train_batches.png）

### 这是什么？
训练过程中的一些样本图片，展示模型在训练数据上的学习效果。

### 怎么看？
- 每张图都是训练时用到的样本
- 上面会标注**边界框（Bounding Box）**和**类别标签**
- 用于直观确认训练数据标注是否正确

---

## 8️⃣ 验证批次对比（val_batches.png）

### 这是什么？
验证集上的**真实标签** vs **模型预测**对比图。

### 怎么看？
- 左图：Ground Truth（真实答案）
- 右图：Prediction（模型预测）
- 如果左右图差不多 → 模型预测准确 ✅
- 如果差很多 → 模型还需要改进

## 9️⃣ 训练参数（training_args.png）

这是训练时用到的一系列配置参数，下面逐一解释：

### 基础参数

| 参数 | 意思 | 建议 |
|------|------|------|
| **task** | 任务类型：detect(检测)、segment(分割)、classify(分类)、pose(姿态) | 目标检测用 detect |
| **mode** | 运行模式：train(训练)、val(验证)、predict(预测) | 训练时用 train |
| **model** | 预训练模型路径，如 yolov8n.pt | 用 yolov8n（nano）最快 |
| **data** | 数据集配置文件路径 | 指向你的 data.yaml |
| **epochs** | 训练轮数，模型看完整数据集的次数 | 一般 50-300，太少学不会，太多过拟合 |
| **batch** | 批次大小，每次看多少张图 | 越大越快，但需要显存够 |
| **imgsz** | 输入图片尺寸 | 一般 640，越大越准但越慢 |

### 优化器参数

| 参数 | 意思 | 建议 |
|------|------|------|
| **lr0** | 初始学习率 | 0.01 左右太大不稳定，太小收敛慢 |
| **lrf** | 最终学习率系数 | 和 lr0 配合控制学习率下降 |
| **momentum** | 动量，帮助加速收敛 | 一般 0.937 |
| **weight_decay** | 权重衰减，防止过拟合 | 一般 0.0005 |

### 正则化参数

| 参数 | 意思 | 建议 |
|------|------|------|
| **patience** | 早停耐心值 | 验证指标连续多少轮没改善就停止，防止白训 |
| **dropout** | Dropout 比率 | 训练时随机丢弃神经元，防止过拟合 |

### 损失函数权重

| 参数 | 意思 | 建议 |
|------|------|------|
| **box** | 边界框损失权重 | 检测框定位准确度 |
| **cls** | 分类损失权重 | 类别分对的重要性 |
| **dfl** | 分布焦点损失权重 | YOLOv8 特有的边框回归 |

### 其他参数

| 参数 | 意思 | 建议 |
|------|------|------|
| **device** | 用哪个设备训练 | 0 是第一块 GPU，cpu 是用 CPU |
| **workers** | 数据加载线程数 | 越大加载越快，但 CPU 占用高 |
| **cache** | 是否缓存数据到内存 | true 加速但内存占用大 |
| **save** | 是否保存训练结果 | 一般 true |
| **plots** | 是否生成可视化图表 | 一般 true |

## 📊 数据从哪里来？

WebUI 右侧面板显示的所有数据，都是从你本地训练的模型文件夹里读取的！

模型文件夹路径：`self-train/models/<你的模型目录>/`

### 文件对应关系表

| WebUI 显示的内容 | 数据来自的文件 | 说明 |
|-----------------|---------------|------|
| **模型性能指标** (mAP@50, Precision, Recall) | `results.csv` | 从 CSV 最后一行的 metrics 列读取 |
| **训练结果图** (results.png) | `results.png` | YOLO 训练自动生成 |
| **混淆矩阵** | `confusion_matrix.png` | YOLO 训练自动生成 |
| **归一化混淆矩阵** | `confusion_matrix_normalized.png` | YOLO 训练自动生成 |
| **Precision-Recall 曲线** | `BoxPR_curve.png` | YOLO 训练自动生成 |
| **训练曲线** (交互式) | `results.csv` | 前端从 CSV 读取数据后用 Recharts 绘制 |
| **训练批次示例** | `train_batch0.jpg`, `train_batch1.jpg`, `train_batch2.jpg` | 训练过程中保存的样本 |
| **验证批次对比** | `val_batch*_labels.jpg`, `val_batch*_pred.jpg` | 验证集标签 vs 模型预测 |
| **训练参数** | `args.yaml` | 训练时的配置文件 |
| **Anchor Box 信息** | 从 `best.pt` / `last.pt` 加载模型后提取 | 代码动态获取 |
| **特征图尺寸** | 从 `best.pt` / `last.pt` 加载模型后提取 | 代码动态获取 |
| **NMS 参数** | 从 `best.pt` / `last.pt` 加载模型后提取 | 代码动态获取 |
| **模型架构** | 从 `best.pt` / `last.pt` 加载模型后提取 | 代码动态获取 |

### 📁 详细文件说明

#### 1. results.csv - 训练指标数据
位置：`self-train/models/session_xxx/results.csv`

这是训练过程中每轮记录的数据，用 Excel 就能打开。

**CSV 列含义：**
| 列名 | 意思 |
|------|------|
| `epoch` | 第几轮 |
| `time` | 本轮训练耗时（秒） |
| `train/box_loss` | 训练集边界框损失 |
| `train/cls_loss` | 训练集分类损失 |
| `train/dfl_loss` | 训练集分布焦点损失 |
| `metrics/precision(B)` | 精确率 |
| `metrics/recall(B)` | 召回率 |
| `metrics/mAP50(B)` | mAP@50 |
| `metrics/mAP50-95(B)` | mAP@50-95 |
| `val/box_loss` | 验证集边界框损失 |
| `val/cls_loss` | 验证集分类损失 |
| `val/dfl_loss` | 验证集分布焦点损失 |
| `lr/pg0`, `lr/pg1`, `lr/pg2` | 各参数组的学习率 |

**示例数据（部分）：**
```
epoch,time,train/box_loss,train/cls_loss,metrics/precision(B),metrics/recall(B),metrics/mAP50(B)
1,3.5,0,112.7,0,0,0
2,5.1,0,110.9,0,0,0
...
50,77.2,0,39.6,0.85,0.82,0.87
```

#### 2. args.yaml - 训练配置
位置：`self-train/models/session_xxx/args.yaml`

这是训练时的所有参数配置，YAML 格式。

**关键参数：**
```yaml
epochs: 50              # 训练轮数
batch: 16               # 批次大小
imgsz: 640               # 输入图片尺寸
device: '0'              # GPU 编号
lr0: 0.01                # 初始学习率
lrf: 0.01                # 最终学习率系数
momentum: 0.937         # 动量
weight_decay: 0.0005    # 权重衰减
box: 7.5                 # 边界框损失权重
cls: 0.5                # 分类损失权重
dfl: 1.5                # 分布焦点损失权重
```

#### 3. 图片文件

| 文件名 | 内容 |
|--------|------|
| `results.png` | 训练过程的损失和指标曲线（matplotlib 生成） |
| `confusion_matrix.png` | 混淆矩阵（样本数量） |
| `confusion_matrix_normalized.png` | 归一化混淆矩阵（比例） |

| `val_batch0_labels.jpg` | 验证集真实标签 |
| `val_batch0_pred.jpg` | 验证集模型预测 |

#### 4. 模型权重文件
位置：`self-train/models/session_xxx/weights/`

| 文件名 | 意思 |
|--------|------|
| `best.pt` | 训练过程中 mAP 最高的模型 |
| `last.pt` | 训练结束时的模型 |

这两个是 PyTorch 模型文件，包含了：
- 模型结构（网络层、参数）
- 训练权重
- Anchor Box 信息
- 类别名称

### 🔄 数据读取流程

```
用户点击模型
    ↓
前端调用 API: /api/models/analysis?model_path=xxx
    ↓
后端 (main.py) 读取对应文件夹：
    1. results.csv → 解析指标数据
    2. args.yaml → 解析训练参数
    3. 各种 .png 图片 → Base64 编码
    4. best.pt → 加载模型提取架构信息
    ↓
返回 JSON 给前端
    ↓
前端渲染展示
```

## 📊 总结：新手应该重点看哪些？

1. **首先看**：模型性能指标（mAP@50、Precision、Recall）
   - mAP@50 > 80% 说明模型基本可用

2. **然后看**：训练曲线
   - 确认 train_loss 和 val_loss 都在下降
   - 如果 val_loss 上升 = 过拟合

3. **最后看**：混淆矩阵
   - 看看哪个类别容易被误判

## ❓ 常见问题

**Q: mAP 很高但实际使用效果不好？**
> 检查 Precision 和 Recall，如果 Recall 低 = 漏检多；如果 Precision 低 = 误检多

**Q: 训练过拟合了怎么办？**
> 增大 patience、减小 epochs、增大 weight_decay、减小 model 尺寸

**Q: 显存不够怎么办？**
> 减小 batch、减小 imgsz

**Q: 找不到某个图表？**
> 检查模型文件夹里有没有对应的文件，如果没有说明训练时可能没开启 `plots: true`
